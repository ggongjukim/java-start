## :round_pushpin: 8. 자바란 ? 
### :white_check_mark: 자파 표준 스펙과 구현
자바는 표준 '스펙' 과 '구현' 두 가지로 나뉘어져있다.
- 1. 자바 표준 스펙 : 자바는 이렇게 만들어야해 표준 스펙 문서
- 2. 구현 : 그 스펙에 맞춰 각 회사들이 JAVA 언어를 만든다 e.g. 오라클, Amazon

<br/>

### :white_check_mark: 컴파일과 실행
자바 프로그램은 '컴파일'과 '실행' 단계를 거친다.
1. 컴파일 :
  - `javac`로 `.java` 파일을 컴파일 과정을 거쳐 `.class` 로 바꾼다 - `javac Hello.java`
  - 변환 할때 바이트코드로 만들어 자바 가상 머신에서 빠르게 실행되도록 최적화,문법오류 검출, 주석제거 등 한다.
2. 실행
  - `java` 프로그램으로 `.class`를 실행해준다 - `java Hello` 
  - `JVM` 이 `.class` 을 읽어서 실행해준다
3. 인텔리제이에서의 컴파일과 실행
- 인텔리제이는 초록색 화살표 누르면 컴파일과 실행을 같이 해준다
- out 이라는 폴더에서 컴파일된 .class 파일들을 확인 가능하다 (decomplie 되어 보인다 원래는 바이트코드)
- out 폴더 내부 파일들을 지워도 run 하면 다시 생긴다

<br/>

### :white_check_mark: 자바와 운영체제 독립성
- java 는 모든 OS 에서 실행가능하다.
- 윈도우, MAC, 리눅스 모두 가능하다.
- 개발할때는 윈도우, MAC 을 쓰고 
- 배포시에는 리눅스 서버를 많이쓴다 
- e.g. Amazon Corretto 자바를 리눅스 서버에 설치하여 사용
